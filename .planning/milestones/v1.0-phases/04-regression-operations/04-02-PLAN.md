# 04-02 Plan: Document Regression Maintenance Workflow

## Objective

<objective>
Document the Phase 4 regression operations workflow in README and align standing
agent/planning docs: explain how to refresh auth state, interpret triage output,
choose local and CI command tiers, update selectors and fixtures, handle sandbox
test data through visible UI only, and preserve cheap developer hooks.
</objective>

## Decision Coverage

- D-09: README preserves the current normal developer hook split.
- D-10: README describes full Playwright CI when storage-state secrets exist.
- D-11: README describes explicit authenticated/full-regression skip behavior.
- D-12: README documents `npm run test:e2e:all` as expected before push when
  valid auth state is available, without enforcing it through `lefthook.yml`.
- D-13: README is the runbook home.
- D-14: README covers auth refresh, triage output, test/fixture updates, and
  command-tier choice.
- D-15: README documents visible UI cleanup only and no hidden cleanup APIs.
- D-16: README documents stable user-visible locators first, test ids only when
  visible locators are insufficient.
- D-01 through D-08: README explains the triage summary, auth/setup section,
  retry/flaky section, and storage-state-first auth failure guidance created in
  Plan 04-01.

  <action>
    Update `README.md` with a new section named exactly:

    `## Regression Maintenance`

    Place it after the `## Commands` section and before `## Project Documents`.

    The section must contain these subsections with exact headings:

    - `### Triage Artifacts`
    - `### Auth And Setup Failures`
    - `### Command Tiers`
    - `### Test And Fixture Updates`
    - `### Sandbox Data Maintenance`

    `### Triage Artifacts` must state:

    - `test-results/triage-summary.md` is the lean operator summary.
    - Native Playwright HTML, JSON, screenshots, traces, videos, and
      `page-errors` remain the authoritative debugging artifacts.
    - Run `npm run test:e2e:triage` to generate the summary from
      `test-results/results.json`.

    `### Auth And Setup Failures` must state:

    - Treat `Auth/Setup State` findings as storage-state problems first.
    - Refresh local auth with `npm run auth:record`.
    - Refresh CI by updating `VERIFYIQ_STORAGE_STATE_JSON` or
      `VERIFYIQ_STORAGE_STATE_PATH`.
    - Do not print credentials, cookies, tokens, or serialized storage state.

    `### Command Tiers` must state:

    - Pre-commit runs `npx lint-staged`.
    - The pre-push hook remains scoped to `npm run check` and
      `npm run test:e2e`.
    - Before pushing when valid auth state is available, run
      `npm run test:e2e:all`.
    - CI runs full Playwright regression when `VERIFYIQ_STORAGE_STATE_JSON` is
      configured.
    - CI writes an explicit authenticated/full-regression skip note when the
      storage-state secret is absent.

    `### Test And Fixture Updates` must state:

    - Use role, label, heading, button, and visible text locators before test
      ids.
    - Use test ids only when visible locators are ambiguous or unavailable.
    - Keep fixtures synthetic and free of real personal, financial, credential,
      cookie, token, or storage-state data.

    `### Sandbox Data Maintenance` must state:

    - Generated records use `AUTOMATION` names.
    - Cleanup must use visible UI controls only.
    - Do not add hidden cleanup API calls.
    - If safe visible cleanup controls are unavailable, document accumulation
      and keep generated names identifiable.

    Also update the `## Commands` list to include:

    - `npm run test:triage` for Node unit tests covering the triage formatter.
    - `npm run test:e2e:triage` for generating the Playwright triage summary.

    Do not modify `docs/ai-development-workflow.md`; Phase 4 runbook content
    belongs in README.
