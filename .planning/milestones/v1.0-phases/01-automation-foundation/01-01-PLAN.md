# 01-01 Plan: Implement Automation Foundation

## Objective

Create the initial TypeScript Playwright automation foundation, including docs,
GSD artifacts, local gates, CI, and smoke tests.

## Tasks

1. Add npm package metadata, TypeScript, ESLint, Prettier, Playwright, Lefthook,
   and lint-staged configuration.
2. Add Playwright config with public, setup, and authenticated projects.
3. Add public root smoke test, auth setup, authenticated smoke test, and initial
   workflow smoke test.
4. Add GitHub Actions workflow for static checks, public smoke, optional
   authenticated tests, and artifact upload.
5. Add `README.md`, `AGENTS.md`, and GSD project docs with cross references.
6. Verify lint, typecheck, docs alignment, and public smoke locally.

## Success Criteria

- `npm run check` passes.
- `npm run test:e2e` passes.
- Auth tests fail clearly when credentials/storage state are missing or
  reCAPTCHA blocks credential login.
- Documentation alignment check passes.
- No secrets or Playwright auth state are committed.

## Threat Model

- Credentials can leak through docs, shell output, storage state, or committed
  fixtures. Mitigation: env-only credentials, ignored `.env*`, ignored
  `playwright/.auth/`, no secret values in docs or source.
- Browser automation can become flaky if selectors are implementation-coupled.
  Mitigation: prefer accessible roles and visible user-facing names.
- Local hooks can become too slow. Mitigation: pre-push runs only static checks
  and public smoke; full auth/regression remains explicit or CI-driven.
