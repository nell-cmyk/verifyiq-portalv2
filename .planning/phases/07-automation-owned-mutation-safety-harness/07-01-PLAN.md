# 07-01 Plan: Build Automation-Owned Record Safety Helpers

## Objective

<objective>
Build a reusable Playwright support harness that creates run-scoped
automation-owned record identities, registers same-run records, fails closed
before unsafe update/delete actions, tracks cleanup residue, and formats
secret-safe diagnostics for Phase 8 workflow specs.

Purpose: prevent future mutating portal tests from updating or deleting
pre-existing VerifyIQ records. Output: `tests/support/automation-records.ts` and
auth-independent Playwright coverage in
`tests/public/automation-records.spec.ts`. </objective>

## Threat Model

<threat_model>

| ID      | Threat                                                                | Severity | Mitigation                                                                                                                        |
| ------- | --------------------------------------------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------- |
| T-07-01 | Future workflow updates or deletes a pre-existing portal record.      | high     | Require context registration plus exact current-run visible-name confirmation before mutation.                                    |
| T-07-02 | Stale `AUTOMATION` records from prior runs are treated as same-run.   | high     | Validate the current run id and exact registered visible name; stale run ids fail with `stale_run_id`.                            |
| T-07-03 | Ambiguous UI matches cause the first matching row to be mutated.      | high     | Fail closed on zero or multiple candidates and return `zero_matches` or `multiple_matches`.                                       |
| T-07-04 | Cleanup failure hides the original test failure or leaves residue.    | high     | Track cleanup notes and aggregate original plus cleanup failures with `AggregateError`.                                           |
| T-07-05 | Harness diagnostics leak secrets or broad page data.                  | high     | Format diagnostics from visible identifiers only: run id, area, visible name, label, route/section, and action.                   |
| T-07-06 | Runner target mappings drift during helper work.                      | medium   | Do not edit runner files; verify `git diff -- scripts/run-portal-automation.mjs scripts/run-portal-automation.test.mjs` is empty. |
| T-07-07 | Helper tests mutate the live portal before Phase 8 defines workflows. | high     | Put helper coverage in public, non-live Playwright tests using fake candidate data and no `page.goto`.                            |

</threat_model>
