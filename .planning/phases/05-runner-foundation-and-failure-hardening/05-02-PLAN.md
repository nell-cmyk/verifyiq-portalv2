- D-11: Playwright failure exit code wins when triage succeeds.
- D-12: Playwright success remains success if triage generation fails.

- D-14: required-applicant validation assertion anchors to inline validation.

## Threat Model

<threat_model>

| ID      | Threat                                                          | Severity | Mitigation                                                                                                                           |
| ------- | --------------------------------------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| T-05-06 | Triage success masks a failing Playwright run.                  | high     | Store Playwright exit code and return it whenever it is non-zero, regardless of triage result.                                       |
| T-05-07 | Triage failure makes a successful Playwright run look failed.   | medium   | Print a warning when triage fails after a successful Playwright run and exit 0.                                                      |
| T-05-08 | Runner output omits native artifact paths needed for debugging. | medium   | Print the four static artifact paths after triage.                                                                                   |
| T-05-09 | Validation assertion still matches toast text.                  | high     | Use `page.getByTestId("validation-error")` or an equivalent inline form surface; do not use page-level `getByText` for this message. |
| T-05-10 | Auth/setup failure recovery text leaks secrets.                 | high     | Preserve existing setup project and triage sanitizer; do not print env values, cookies, tokens, or storage JSON from the runner.     |

</threat_model>
