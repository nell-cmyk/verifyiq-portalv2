# 05-01 Plan: Build Playwright-Backed Runner Foundation

## Objective

<objective>
Create the unified portal runner command surface without moving browser
automation out of Playwright Test. The plan adds `npm run test:portal`, a thin
Node runner that validates target names and builds native Playwright CLI args,
and unit tests that prove command mapping, passthrough behavior, and invalid
target handling before any browser process starts.

Purpose: Establish the operator-facing entrypoint and stable v1.1 target
contract while preserving Playwright as the executable source of truth. Output:
`scripts/run-portal-automation.mjs`, `scripts/run-portal-automation.test.mjs`,
and updated `package.json` scripts. </objective>

## Decision Coverage

- D-01: add `npm run test:portal`.
- D-02: default target is `all`.
- D-03: pre-wire the full v1.1 valid target allowlist.
- D-04: target syntax is `npm run test:portal -- <target>`.
- D-05: future deep targets may initially map to current safe auth coverage.
- D-06: print the underlying Playwright command before execution.
- D-07: unknown targets fail before Playwright starts and print valid targets.
- D-08: debug flags are accepted only after a `--` separator.
- D-09: runner delegates to the local Playwright binary and does not import
  browser automation APIs.

## Threat Model

<threat_model>

| ID      | Threat                                                            | Severity | Mitigation                                                                                                                          |
| ------- | ----------------------------------------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| T-05-01 | Runner prints secret-bearing environment values or storage state. | high     | Do not echo `process.env`, `.env`, `playwright/.auth/`, or storage-state values; print only command args and static artifact paths. |
| T-05-02 | Runner becomes a parallel browser framework.                      | high     | Runner must spawn local Playwright CLI; it must not import `chromium`, `page`, `test`, or browser APIs from Playwright.             |
| T-05-03 | Unknown or misspelled targets run unintended coverage.            | medium   | Validate against the exact allowlist and exit before spawning Playwright.                                                           |
| T-05-04 | Passthrough parsing sends debug flags as target names.            | medium   | Only treat args after an explicit `--` separator as Playwright passthrough flags.                                                   |
| T-05-05 | Future deep targets imply coverage that does not exist yet.       | medium   | Map `activity`, `audit-logs`, `users`, and `roles` to current authenticated coverage only; do not add deep assertions in Phase 5.   |

</threat_model>

  <action>
    Create `scripts/run-portal-automation.mjs` as an ESM Node script with no new
    package dependencies.

    Export these exact constants and functions for unit tests:

    - `export const VALID_TARGETS = ["all", "public", "auth", "applications", "activity", "audit-logs", "users", "roles"];`
    - `export const ARTIFACT_PATHS = ["test-results/triage-summary.md", "playwright-report/", "test-results/results.json", "test-results/artifacts/"];`
    - `export function parsePortalArgs(argv)`
    - `export function buildPlaywrightArgs(target, passthroughArgs = [])`
    - `export function formatPlaywrightCommand(playwrightBin, args)`
    - `export function formatValidTargets()`

    `parsePortalArgs(argv)` must implement this grammar:

    - `[]` returns `{ target: "all", passthroughArgs: [] }`.
    - `["applications"]` returns `{ target: "applications", passthroughArgs: [] }`.
    - `["applications", "--", "--headed"]` returns `{ target: "applications", passthroughArgs: ["--headed"] }`.
    - `["--", "--headed"]` returns `{ target: "all", passthroughArgs: ["--headed"] }`.
    - Unknown target strings return an invalid result that includes the invalid
      target and the allowlist; Playwright must not be spawned for invalid
      results.

    `buildPlaywrightArgs(target, passthroughArgs)` must return these exact base
    Playwright args before appending passthrough args:

    - `all`: `["test"]`
    - `public`: `["test", "--project=public-smoke"]`
    - `auth`: `["test", "--project=authenticated-chromium"]`
    - `applications`: `["test", "--project=authenticated-chromium", "tests/authenticated/add-application.spec.ts"]`
    - `activity`: `["test", "--project=authenticated-chromium"]`
    - `audit-logs`: `["test", "--project=authenticated-chromium"]`
    - `users`: `["test", "--project=authenticated-chromium"]`
    - `roles`: `["test", "--project=authenticated-chromium"]`

    Add a runner entrypoint guarded by `if (process.argv[1] && import.meta.url === new URL(process.argv[1], "file:").href)` or an equivalent ESM-safe check. The entrypoint must resolve the local Playwright binary from `node_modules/.bin/playwright` using `playwright.cmd` on Windows and `playwright` on other platforms.

    For this task, the entrypoint may only validate, print the underlying
    Playwright command, and spawn Playwright with inherited stdio. Triage
    integration and final exit-code policy are completed in Plan 05-02.
