## Threat Model

<threat_model>

| ID      | Threat                                                                                   | Severity | Mitigation                                                                                                                              |
| ------- | ---------------------------------------------------------------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| T-08-04 | Users workflow updates, assigns, deactivates, deletes, or cleans up a pre-existing user. | high     | Create and register a same-run user, collect visible candidates, and call `assertSameRunMutationTarget` before each mutation.           |
| T-08-05 | Users role assignment uses a pre-existing role as mutation proof.                        | high     | Create and register a same-run role inside the Users test before role assignment; fail closed if the UI cannot isolate that role.       |
| T-08-06 | User creation sends a real email invite or requires a deliverable address.               | high     | Inspect invite/email controls first, use only `example.invalid` synthetic addresses when accepted, and stop if risk is uncontrolled.    |
| T-08-07 | Roles workflow edits or deletes a pre-existing role.                                     | high     | Create and register a same-run role, collect visible role candidates, and guard edit/delete with `assertSameRunMutationTarget`.         |
| T-08-08 | Permission toggles change broad access state and are not restored.                       | high     | Include permission toggles only when live inspection identifies a reversible low-risk toggle; otherwise omit and document the reason.   |
| T-08-09 | Cleanup failures hide original workflow failures or leak raw cleanup details.            | high     | Use `recordAutomationCleanup`, `getAutomationCleanupFailures`, and `aggregateAutomationFailures` with bounded visible identifiers only. |
| T-08-10 | Auth-state setup output leaks storage state or credentials.                              | high     | Use existing auth setup errors and never print credential values, cookies, tokens, serialized storage state, or `.env` values.          |

</threat_model>
