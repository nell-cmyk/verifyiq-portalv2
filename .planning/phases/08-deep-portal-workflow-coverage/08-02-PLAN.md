## Threat Model

<threat_model>

| ID      | Threat                                                                        | Severity | Mitigation                                                                                                                              |
| ------- | ----------------------------------------------------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| T-08-04 | Users workflow updates, assigns, removes, or cleans up a pre-existing user.   | high     | Create and register a same-run user, collect visible candidates, and call `assertSameRunMutationTarget` before every mutation.          |
| T-08-05 | Users role assignment uses a pre-existing role as mutation proof.             | high     | Create and register a same-run role inside the Users test before role assignment; fail closed if the UI cannot isolate that role.       |
| T-08-06 | User creation sends a real email invite or requires a deliverable address.    | high     | Use the inspected password-based create flow and only `example.invalid` synthetic email addresses.                                      |
| T-08-07 | Roles workflow edits or deletes a pre-existing role.                          | high     | Create and register a same-run role, collect visible role candidates, and guard delete with `assertSameRunMutationTarget`.              |
| T-08-08 | Role edit coverage is falsely implemented despite no visible edit action.     | high     | Do not export `updateAutomationRole`; annotate the `@portal:roles` spec and summary with the MUT-07 role edit product blocker.          |
| T-08-09 | Cleanup failures hide original workflow failures or leak raw cleanup details. | high     | Use `recordAutomationCleanup`, `getAutomationCleanupFailures`, and `aggregateAutomationFailures` with bounded visible identifiers only. |
| T-08-10 | Auth-state setup output leaks storage state or credentials.                   | high     | Use existing auth setup errors and never print credential values, cookies, tokens, serialized storage state, or `.env` values.          |
| T-08-14 | Runner target mappings drift to compensate for cross-page setup.              | medium   | Keep setup inside target-owned tests and verify runner mapping files are unchanged.                                                     |

</threat_model>
