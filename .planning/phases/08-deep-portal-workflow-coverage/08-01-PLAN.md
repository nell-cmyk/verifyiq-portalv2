<objective>

polluting runner target mappings. Output: an evidence helper plus

## Threat Model

<threat_model>

| ID      | Threat                                                                                          | Severity | Mitigation                                                                                                                                                                    |
| ------- | ----------------------------------------------------------------------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-08-01 | Activity coverage passes by checking generic page tooling instead of same-run evidence.         | high     | Generate a same-run Users/Roles action inside the `@portal:activity` test and require exact marker or run id visibility.                                                      |
| T-08-02 | Audit Logs coverage mutates log data or weakens matching to any recent row.                     | high     | Never mutate Audit Logs directly; generate same-run evidence elsewhere and match exact full marker or run id with bounded polling.                                            |
| T-08-03 | Activity/Audit setup updates or deletes pre-existing Users/Roles records.                       | high     | Reuse plan 08-02 helpers that register same-run records and call `assertSameRunMutationTarget` before mutation or cleanup.                                                    |
| T-08-11 | Asynchronous log arrival causes flaky failures or false passes.                                 | medium   | Use bounded `expect.poll` intervals from the live inspection and never accept a row without same-run marker or run id proof.                                                  |
| T-08-12 | Target pollution causes `activity` or `audit-logs` runner targets to execute Users/Roles specs. | medium   | Keep setup inside target-owned specs and do not change runner grep mappings.                                                                                                  |
| T-08-13 | Evidence or diagnostics leak secret-bearing auth data.                                          | high     | Attach only visible automation identifiers, route/section, timing, and bounded blocker reasons; no credentials, cookies, tokens, storage state, raw DOM, or network payloads. |

</threat_model>
