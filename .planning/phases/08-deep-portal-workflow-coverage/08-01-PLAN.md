## Threat Model

<threat_model>

| ID      | Threat                                                                                          | Severity | Mitigation                                                                                                                |
| ------- | ----------------------------------------------------------------------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------- |
| T-08-01 | Activity coverage passes by checking generic page tooling instead of same-run evidence.         | high     | Generate a same-run role action inside the `@portal:activity` test and require exact marker or run id visibility.         |
| T-08-02 | Audit Logs coverage falsely claims same-run portal activity evidence.                           | high     | Add export-control coverage plus a `test.fixme` blocker; do not assert generic exports as MUT-05 success.                 |
| T-08-03 | Activity setup updates or deletes pre-existing Roles records.                                   | high     | Reuse plan 08-02 role helpers that register same-run records and call `assertSameRunMutationTarget` before cleanup.       |
| T-08-11 | Asynchronous Activity arrival causes flaky failures or false passes.                            | medium   | Use bounded polling and never accept a row without same-run marker or run id proof.                                       |
| T-08-12 | Target pollution causes `activity` or `audit-logs` runner targets to execute Users/Roles specs. | medium   | Keep setup inside target-owned specs and do not change runner grep mappings.                                              |
| T-08-13 | Evidence or diagnostics leak secret-bearing auth data.                                          | high     | Attach only visible automation identifiers, route/section, timing, and bounded blocker reasons.                           |
| T-08-15 | Future product changes make Audit Logs evidence available but tests keep skipping MUT-05.       | medium   | Name the blocker in a `test.fixme` with `MUT-05`; when product exposes evidence, remove the fixme and add exact matching. |

</threat_model>
