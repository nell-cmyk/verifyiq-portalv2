# 04-01 Plan: Add Secret-Safe Playwright Triage Output

## Objective

<objective>
Add an operator-friendly Playwright triage summary without replacing native
Playwright artifacts: generate `test-results/triage-summary.md`, classify
auth/setup state separately from application failures, surface retry-pass/flaky
tests, preserve local retry and trace policies, and wire CI so full regression
runs only when authenticated storage state is available.
</objective>

## Context

- `.planning/phases/04-regression-operations/04-CONTEXT.md`
- `.planning/phases/04-regression-operations/04-RESEARCH.md`
- `.planning/phases/04-regression-operations/04-VALIDATION.md`
- `.planning/phases/04-regression-operations/04-PATTERNS.md`
- `AGENTS.md`
- `package.json`
- `playwright.config.ts`
- `.github/workflows/e2e.yml`

## Decision Coverage

- D-01: implement the smallest useful triage improvement as a lean Markdown
  summary that complements native Playwright artifacts.
- D-02: write `test-results/triage-summary.md` locally and upload it through CI
  artifact handling.
- D-03: format only allowlisted report fields and sanitize secret-like text.
- D-04: include a separate auth/setup state section.
- D-05: preserve `retries: process.env.CI ? 2 : 0`.
- D-06: include retry/flaky section for tests that pass after retry.
- D-07: preserve `trace: "on-first-retry"`.
- D-08: classify auth-state failures with storage-state-first recovery guidance.
- D-10: run full Playwright regression in CI when storage-state JSON is present.
- D-11: explicitly record a skip when authenticated/full regression cannot run.

## Threat Model

<threat_model>

| ID         | Threat                                                                                          | Severity | Mitigation                                                                                                                              |
| ---------- | ----------------------------------------------------------------------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| T-04-01-01 | Triage output leaks credentials, cookies, tokens, request headers, or serialized storage state. | high     | Use allowlisted JSON report fields, sanitize error text, add synthetic secret-leak tests, and never read `.env` or `playwright/.auth/`. |
| T-04-01-02 | Expired auth state is mislabeled as a VerifyIQ app regression.                                  | high     | Classify setup project, `auth.setup.ts`, and known auth-state messages under `Auth/Setup State`.                                        |
| T-04-01-03 | Retry-pass tests disappear inside green CI.                                                     | medium   | Add a `Retry/Flaky Tests` section whenever Playwright reports `status: "flaky"` or any result has `retry > 0`.                          |
| T-04-01-04 | CI fails forks or unauthenticated environments by requiring auth secrets.                       | high     | Keep auth-dependent full regression behind `VERIFYIQ_STORAGE_STATE_JSON != ''` and write an explicit skip note when absent.             |
| T-04-01-05 | Operational changes replace native Playwright artifacts.                                        | medium   | Preserve HTML, JSON, screenshots, traces, videos, and upload paths; summary links operators back to native artifacts.                   |

</threat_model>

  </read_first>
  <behavior>
    - Given a synthetic Playwright JSON report with one setup failure, `formatSummary` includes `## Auth/Setup State` and the known storage-state recovery command.
    - Given a synthetic report with one application failure, `formatSummary` includes `## Failed Application Tests` with project, file, title, status, retry count, and sanitized error message.
    - Given a synthetic report with `status: "flaky"` or a result where `retry > 0`, `formatSummary` includes `## Retry/Flaky Tests`.
    - Given synthetic stdout/stderr or errors containing `redacted-test-value`, `VERIFYIQ_PASSWORD=secret`, `cookie: abc`, `token: abc`, or `storageState`, generated output does not contain those raw values.
  </behavior>
  <action>
    Create `scripts/summarize-playwright-results.mjs` as an ESM Node script with
    no new package dependencies.

    Export these concrete functions for tests:

    - `export function sanitizeText(value)`
    - `export function collectTests(suites)`
    - `export function classifyTest(test, spec)`
    - `export function formatSummary(report, options = {})`
    - `export async function writeSummary({ resultsPath = "test-results/results.json", outputPath = "test-results/triage-summary.md", skipNotePath = "test-results/auth-regression-skip.md" } = {})`

    `collectTests` must recursively walk `report.suites`, nested `suite.suites`,
    `suite.specs`, `spec.tests`, and `test.results`.

    `classifyTest` must return one of these categories:

    - `auth_setup` when `test.projectName === "setup"`, `spec.file` contains
      `auth.setup.ts`, any error message contains `Stored VerifyIQ auth state
      from`, or any error message contains `Invalid VERIFYIQ_STORAGE_STATE_JSON`.
    - `retry_flaky` when `test.status === "flaky"` or any result has
      `retry > 0`.
    - `application_failure` when `test.status === "unexpected"` and it is not
      `auth_setup`.
    - `skipped` when `test.status === "skipped"`.
    - `expected` otherwise.

    `sanitizeText` must redact at least these patterns:

    - `VERIFYIQ_USERNAME=...`
    - `VERIFYIQ_PASSWORD=...`
    - `VERIFYIQ_STORAGE_STATE_JSON=...`
    - `VERIFYIQ_STORAGE_STATE_PATH=...`
    - `cookie: ...`
    - `token: ...`
    - `authorization: ...`
    - `storageState: ...`
    - `redacted-test-value`

    `formatSummary` must write Markdown with these exact headings:

    - `# Playwright Triage Summary`
    - `## Run Stats`
    - `## Auth/Setup State`
    - `## Failed Application Tests`
    - `## Retry/Flaky Tests`
    - `## Skipped Tests`
    - `## Native Artifacts`

    Include only allowlisted fields in row output: project name, file, test
    title, final status, retry numbers, duration, attachment names, and
    sanitized error message. Do not include raw stdout/stderr bodies unless they
    are passed through `sanitizeText` and truncated to 300 characters.

    `writeSummary` must create `test-results/` when needed. If
    `test-results/results.json` is missing, it must still write
    `test-results/triage-summary.md` with `## Run Stats` and the exact text
    `No Playwright JSON report found at test-results/results.json.` and exit 0.
    If `test-results/auth-regression-skip.md` exists, include its sanitized
    content in `## Auth/Setup State`.

    Add `scripts/summarize-playwright-results.test.mjs` using `node:test` and
    `node:assert/strict` to cover the four behavior bullets above.

    Update `package.json` scripts:

    - Add `"test:triage": "node --test scripts/summarize-playwright-results.test.mjs"`.
    - Add `"test:e2e:triage": "node scripts/summarize-playwright-results.mjs"`.
    - Update `check` to run `npm run lint && npm run typecheck && npm run test:triage && npm run docs:check`.

  </action>
